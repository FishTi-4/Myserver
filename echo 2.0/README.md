# 简单 TCP 行协议服务器/客户端

一个轻量级的 C++ 项目，实现了一个多客户端 TCP 服务器和一个配套的客户端，采用简单的基于行的文本协议。服务器使用 `epoll` 实现高性能事件驱动 I/O，支持边缘触发模式，能处理多个并发连接。服务器和客户端均在 Linux 环境下运行。

## 特性

- 基于 `epoll`（ET 模式）的非阻塞 TCP 服务器
- 行导向协议：消息以 `\n` 分隔
- 支持多个客户端同时连接
- 按连接自动管理接收缓冲区
- 优雅处理客户端断开和错误
- 简单的交互式客户端，发送用户输入并打印服务器响应

## 环境要求

- Linux 操作系统
- 支持 C++11 的编译器（如 g++）
- 标准 POSIX 套接字 API 和 `epoll`（现代 Linux 发行版均内置）

## 使用方法

1. 启动服务器，指定监听端口：

   ```bash
   ./server <端口号>
   ```

   示例：
   ```bash
   ./server 8080
   ```

2. 启动一个或多个客户端，指定服务器的 IP 地址和端口：

   ```bash
   ./client <IP地址> <端口号>
   ```

   示例（连接本地服务器）：
   ```bash
   ./client 127.0.0.1 8080
   ```

3. 在客户端输入消息并按回车键，服务器会返回带有确认前缀的相同消息。输入 `exit` 可关闭客户端连接。

## 协议说明

- 每条消息为单行文本，以换行符（`\n`）结尾。
- 服务器读取数据，直到遇到 `\n`，然后处理完整的一行。
- 当服务器收到一行消息后，回复：  
  `服务器已收到消息: <原始消息>\n`
- 如果客户端发送 `exit`，客户端会关闭连接（但仍会先发送 `exit\n` 给服务器）。

## 实现细节

- 使用 `AF_INET`、`SOCK_STREAM | SOCK_NONBLOCK` 在指定端口上创建套接字并进行绑定。
- 监听队列长度为 5。
- 使用 `epoll_create1(0)` 创建 epoll 实例，并将监听套接字以 `EPOLLIN | EPOLLET` 事件添加。
- 新连接到来时，使用 `accept4(…, SOCK_NONBLOCK)` 接受，并将新的客户端文件描述符加入 epoll。
- 通过 `std::map<int, std::string>` 维护每个套接字的接收缓冲区，并使用 `std::array<char,1024>` 作为临时读取缓冲区。
- `get_date::gt_lin()` 函数实现行提取：
  - 若缓冲区中已存在完整行（包含 `\n`），则将其切分返回。
  - 否则，从套接字读取更多数据并追加到缓冲区。
  - 返回值：`1`（得到一行），`0`（尚无完整行，需等待），`2`（发生错误或连接关闭）。
- 发生错误或客户端断开时，从 epoll 中删除对应的文件描述符并关闭。

### 客户端（`ctcpclient`）
- 创建一个阻塞 TCP 套接字并连接到服务器。
- 从 `std::cin` 读取用户输入，添加 `\n` 后通过 `write()` 发送。
- 采用类似的行接收方式：缓冲数据直到遇到 `\n`，然后显示服务器响应。

## 注意事项

- 服务器使用 `errno` 的 `EAGAIN` / `EWOULDBLOCK` 值来正确判断非阻塞就绪状态。
- 服务器运行在单线程中，使用 epoll 驱动，没有显式的线程安全机制。
- 存储连接缓冲区的映射表仅在客户端断开时才被清理，对于中等数量的客户端是可以接受的。
- 如果服务器未发送完整行，客户端的接收循环可能会无限阻塞，此设计仅为简单起见。
- 请确保防火墙允许所选端口，或仅在本地测试。
